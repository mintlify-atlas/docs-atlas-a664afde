The InfoJobs DevBoard repository is organized as a monorepo where each top-level numbered directory represents a self-contained implementation stage. Each stage builds on concepts introduced in the previous one, taking the project from a static HTML/CSS prototype through vanilla JavaScript, React, routing, state management, a REST API, TypeScript, AI integration, and finally a SQL-backed backend. You do not need to work through every stage вҖ” most readers will want to go straight to `07-inteligencia-artificial`, which is the complete full-stack application.

## Top-level layout

```
infoJobs/
в”ңв”Җв”Җ 00-html-css/                  # Static HTML/CSS prototype
в”ңв”Җв”Җ 01-javascript/                # Vanilla JS with local JSON data
в”ңв”Җв”Җ 02-react/                     # React 19 with a custom router
в”ңв”Җв”Җ 02-react-cdn-version/         # React via CDN (no build step)
в”ңв”Җв”Җ 03-router-and-zustand/        # React Router DOM 7 + Zustand

в”ңв”Җв”Җ 06-typescript/                # TypeScript fundamentals
```

## Implementation stages

| Stage | Directory | Description |
|---|---|---|
| 0 | `00-html-css` | Pure HTML/CSS prototype вҖ” static markup and styling with no JavaScript |
| 1 | `01-javascript` | Vanilla JS that fetches job data from a local JSON file and renders listings dynamically |
| 2 | `02-react` | React 19 application built with a custom hand-rolled router (no React Router) |
| 3 | `03-router-and-zustand` | Adds React Router DOM 7 for client-side navigation and Zustand 5 for global state |
| 4 | `04-express` | Express 5 REST API in JavaScript with CRUD endpoints, Zod validation, and CORS |
| 5 | `04-node` | Node.js fundamentals: CLI scripts, the `fs` module, and a raw HTTP server |
| 6 | `05-testing` | Playwright end-to-end test suite covering the job board UI |
| 7 | `06-typescript` | TypeScript language fundamentals applied to the project codebase |
| 8 | `07-inteligencia-artificial` | **Main app** вҖ” React 19 frontend + Express 5 backend + Ollama AI summaries |
| 9 | `08-sql` | TypeScript backend using Express and `better-sqlite3` for persistent SQLite storage |
